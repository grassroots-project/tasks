## 人才库使用流程

### 作为新成员

#### 第一步：加入人才库

**条件**：
- 已通过筛选问卷
- 已被邀请加入项目
- 了解并接受项目价值观

**操作**：
1. 访问人才库：data/people.md
2. Fork 该仓库
3. 在"现有成员"部分添加你的信息：

```markdown
### [你的名字/昵称]

- **加入时间**：2026-MM-DD
- **技能标签**：[用逗号分隔，例如：文案、设计、技术]
- **时间承诺**：每周 Xh
- **当前任务**：[任务编号和名称，或任务链接]
- **历史贡献**：[列出你做过的任务]
```

4. 提交 Pull Request
5. 等待合并

#### 第二步：认领第一个任务

**目的**：
- 熟悉任务池系统
- 建立协作习惯
- 开始贡献记录

**操作**：
1. 访问任务池：https://www.grassroots-project.app/pages/tasks.html
2. 选择一个你感兴趣的任务
3. 在该任务 Issue 中评论：`@你的昵称 认领这个任务`
4. 将任务标签从 `待领` 改为 `进行中`
5. 更新人才库中的"当前任务"

#### 第三步：记录贡献

**操作**：
1. 完成任务后，更新任务 Issue 的状态为 `已完成`
2. 在人才库的"历史贡献"中添加该任务
3. 更新"当前任务"或设为"无"

## 资源池使用流程

### 作为使用者

#### 第一步：查看可用资源

**操作**：
1. 访问资源池：data/resources.md
2. 查看"资源列表"部分
3. 找到你想用的资源

#### 第二步：申请使用资源

**操作**：
1. 确认资源状态为"可用"
2. 查看资源负责人的联系方式
3. 与负责人沟通使用需求
4. 等待确认

**沟通内容应包括**：
- 你是谁（人才库中的信息）
- 你想用什么资源
- 用途是什么
- 预期使用多久/多少

#### 第三步：使用资源

**操作**：
1. 按照使用说明使用资源
2. 遵守资源使用的注意事项
3. 使用后及时反馈

#### 第四步：更新资源状态

**重要**：使用后更新资源状态

**操作**：
1. Fork 仓库
2. 编辑 data/resources.md
3. 更新资源的"当前状态"
   - 从"可用"改为"已占用"（如果正在用）
   - 从"已占用"改为"可用"（用完后）
4. 提交 PR
5. 合并 PR

---

### 作为资源负责人

#### 管理资源

**操作**：
1. 接收到资源使用请求
2. 评估请求的合理性
3. 确认资源可用
4. 批准或拒绝请求
5. 指导使用者正确使用资源
6. 使用后确认资源归还

#### 更新资源信息

**何时需要更新**：
- 资源信息变化
- 资源状态变化
- 新增资源

**操作**：
1. Fork 仓库
2. 编辑 data/resources.md
3. 更新资源信息
4. 提交 PR
5. 合并 PR

## 三池协同工作流程

### 场景 1：新成员加入并认领第一个任务

```
1. 新成员通过筛选问卷
   ↓
2. 新成员加入人才库（data/people.md）
   ↓
3. 新成员查看任务池（网站）
   ↓
4. 新成员认领任务（任务 Issue）
   ↓
5. 新成员使用资源（data/resources.md，如果需要）
   ↓
6. 新成员完成任务
   ↓
7. 更新人才库（历史贡献）
   ↓
8. 更新任务状态
   ↓
9. 更新资源状态（如果使用了）
```

### 场景 2：团队协作完成一个任务

```
1. 在任务池创建新任务
   ↓
2. 成员 A 认领主任务
   ↓
3. 成员 B 和 C 协作（通过资源池沟通）
   ↓
4. A 使用 AI 工具（资源池）
   ↓
5. A、B、C 共同完成任务
   ↓
6. 更新所有人的历史贡献（人才库）
   ↓
7. 关闭任务（任务池）
```

## 常见问题

### Q1：我没有 GitHub 怎么办？

**A**：现在就需要 GitHub，因为我们的任务池、人才库、资源池都基于 GitHub。

**如何创建 GitHub 账号**：
1. 访问：https://github.com
2. 点击 "Sign up"
3. 选择注册方式（邮箱、GitHub App、邮箱+密码等）
4. 完成注册

### Q2：我需要 Fork 整个仓库吗？

**A**：不一定。根据你的使用方式：

**需要 Fork 的情况**：
- 你要频繁编辑人才库/资源池
- 你要贡献多个 PR
- 你想自己维护副本

**不需要 Fork 的情况**：
- 你只是偶尔认领任务
- 你只是使用资源（如 AI）

**替代方案**：
- 直接在 Issue 中评论
- 发邮件给维护者更新

### Q3：人才库和任务池中的信息如何同步？

**A**：需要手动同步。

**当前系统的同步关系**：

| 池子 | 更新方式 | 需要同步的信息 |
|------|---------|---------------|
| 人才库 | 手动编辑 data/people.md + PR | 成员信息、技能、当前任务、历史贡献 |
| 任务池 | 手动编辑 Issue 或使用网站 | 任务状态、标签 |
| 资源池 | 手动编辑 data/resources.md + PR | 资源状态、使用记录 |

**未来可能**：
- 使用 GitHub Actions 自动同步
- 使用 Webhook 自动触发更新
- 使用第三方工具集成

### Q4：如何知道我的任务进度被看到了？

**A**：
1. 任务池网站会显示所有任务
2. 看板页面会显示任务进度
3. 人才库中的"任务参与情况"会更新

### Q5：我可以用假名字/昵称吗？

**A**：可以。只要在一个地方保持一致即可。

**建议**：
- 在 GitHub 使用统一的昵称
- 在人才库中使用相同的昵称
- 这样更容易追踪贡献

## 最佳实践

### 1. 定期更新信息

**人才库**：
- 至少每周更新一次"当前任务"
- 完成任务后立即更新"历史贡献"
- 技能或时间承诺变化时更新

**资源池**：
- 使用资源前先沟通
- 使用后立即更新状态
- 定期检查资源信息的准确性

### 2. 保持信息一致性

- 确保人才库、任务池、资源池中的信息一致
- 如果更新了某个地方，记得同步更新其他地方
- 特别是任务编号和链接

### 3. 及时沟通

- 使用资源前先沟通
- 有问题及时问负责人
- 重要决定在 Issues 中讨论（群聊容易丢失）

### 4. 尊重他人时间

- 申请资源时要说明清楚用途和预期时间
- 不要长期占用稀缺资源
- 用完后及时归还
